git init  => Initialise the Git monitoring on that folder
git clone  => To clone the remote repository into the local repo.
git status  => To know the status of the files in the local repo i.e. any changes done, are changed files commited or ready to commit.
git add -A  => To push the changes we made to the Staging. This will push all files in the local repo.
git add . => This is also used to push the changes done into the staging but it only pushes the files in the current directory on which this command is executed.
git commit -m <commit message>  => Used to commit the changes.
git reset HEAD~  => This will reset all changes done and moves the files from the staging to the local working directory i.e. into unmonitor state.
git rm <file-name>  => Un-stages and deletes the file.
git reset --hard  => This will resets all changes done along with bringing back the deleted files.
git rm --cached  => Deletes the file from the staging but not from the local repo.

<<From Staging>>

git log => To check the commit history and other logs
git log --oneline  => Prints the same but in short and simple.
git branch => lists all available branch and you see * infront of branch name which indicates that you are currently on that branch
git branch <Branch Name>  => Creates the new branch.
git checkout <Branch Name>  => Switches to that branch.
git merge <Sub-Branch> -m "Merging into Main"  => This will merges the changes done on the sub-branch into the main branch (this command need to be executed by switching into the main branch)
git push origin <Branch-Name>  => Pushes the changes committed on the local repo onto that specific branch.
git stash => Saves the un-committed changes to a temporary location.
git stash list  => Lists all stashed data
git stash pop  => Reverts the saved data and deletes it from the stash queue.
git stash apply  => Reverts the saved data but doesn't delete it from the stash queue.
git stash drop  => Deletes the saved data from the queue.
